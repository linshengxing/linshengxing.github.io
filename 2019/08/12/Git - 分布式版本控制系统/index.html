Git-分布式版本控制系统

发表于 2019-08-12 | 更新于 2019-08-13 | 评论数： | 热度： ℃

配置用户名和Email地址

配置用户名：$ git config –global user.name “Your Name”
配置Email地址：$ git config –global user.email “email@example.com“
查看用户名：$ git config user.name
查看Email地址：$ git config user.email

创建版本库

创建空仓库：$ mkdir learngit
改变目录路径：$ cd learngit
显示当前目录：$ pwd
将该目录变成Git管理的仓库：$ git init
仓库创建完，在当前目录下多了一个.git的目录，这个目录是Git用来跟踪管理版本库的
Windows系统，为了避免遇到各种莫名其妙的问题，请确保目录名（包括父目录）不包含中文。

将文件添加到版本库

版本控制系统，只能跟踪文本文件的改动，图片、视频是二进制文件，版本控制系统无法跟踪文件的变化，只知道图片从100KB改成120KB，具体改了什么，并不知道。（Word是二进制格式）
在仓库learngit目录下编写一个readme.txt文件
Git is a version control system.
Git is free software.
将文件添加到仓库：$ git add readme.txt
将文件提交到仓库：$ git commit -m “wrote a readme file”
为什么Git添加文件需要add、commit两步？因为commit可以提交多个文件

修改文件

修改readme.txt文件内容
Git is a distributed version control system.
Git is free software.
查看仓库当前状态：$ git status
查看修改的内容：$ git diff

版本回退

查看提交历史：$ git log
查看提交历史（简短）：$ git log –pretty=oneline
回退到上一个版本：$ git reset –hard HEAD^
回退到指定版本：$ git reset –hard commit_id
查看命令历史：$ git reflog
commit_id（版本号），是一个SHA1计算出来的非常大的数字，用十六进制表示。

工作区和暂存区

工作区（Working Directory），就是你在电脑里面能看到的目录。
工作区有一个隐藏目录.git，这个不算工作区，而是Git的版本库（Repository）。
Git的版本库里存了很多东西，其中最重要的就是称为stage（或者叫index）的暂存区，还有Git为我们自动创建的第一个分支master，以及指向master的一个指针叫HEAD。

【git add】把文件添加到仓库，实际上就是把文件修改添加到暂存区。
【git commit】把文件提交到仓库，实际上就是把暂存区的所有内容提交到当前分支。

管理修改

Git跟踪并管理的是修改，而非文件。每次修改，如果不用【git add】添加到暂存区，那就不会加入到【git commit】中。
查看工作区和版本库里面最新版本的区别：$ git diff HEAD – readme.txt

撤销修改

撤销工作区的修改：$ git checkout – readme.txt
撤销修改分两种情况：
一种是readme.txt自修改后还没有被放到暂存区，撤销修改回到和版本库一模一样的状态。
一种是readme.txt已经添加到暂存区，又作了修改，撤销修改回到添加到暂存区后的状态。
将暂存区的修改撤销掉（unstage），放回工作区：$ git reset HEAD readme.txt

删除文件

删除工作区的文件：$ rm test.txt
删除版本库的文件：$ git rm test.txt 删除后需提交：$ git commit -m “remove test.txt”
还原版本库的文件到工作区：$ git checkout – test.txt

远程仓库

创建SSH Key：$ ssh-keygen -t rsa -C “youremail@example.com“
在电脑用户主目录下生成.ssh目录，id_rsa是私钥（不要泄露），id-rsa.pub是公钥匙

添加远程库

关联ssh协议远程仓库：$ git remote add origin git@github.com:linshengxing/repository.git
通过https关联远程仓库：$ git remote add origin https://github.com/linshengxing/repository.git
第一次推送master分支的所有内容：$ git push -u origin master
推送本地内容到远程仓库：$ git push origin master
通过ssh克隆远程内容到本地仓库：$ git clone git@github.com:linshengxing/repository.git
通过https克隆远程内容到本地仓库：$ git clone https://github.com/linshengxing/repository.git
删除GitHub上的远程库：$ git remote rm origin
由于远程库是空的，第一次推送master分支时，加上-u参数，Git不但会把本地的master分支内容推送到远程新的master分支，还会把本地的master分支和远程的master分支关联起来。
Git支持多种协议，默认的git://使用ssh，也可以使用https。
使用https除了速度慢以外，还有每次推送都必须输入口令，但是在某些只开放http端口的公司内部就只能使用https协议。

创建与合并分支

查看分支：$ git branch
创建分支：$ git branch dev
切换分支：$ git checkout dev
创建+切换分支：$ git checkout -b dev
合并某分支到当前分支：$ git merge dev
删除分支：git branch -d dev

冲突解决

Git用<<<<<<< ，======= ，>>>>>>> 标记出不同分支的内容。
查看分支的合并情况：$ git log –graph –pretty=oneline –abbrev-commit

分支管理策略

通常合并分支时，Git会用Fast forward模式，在这种模式下，删除分支后，会丢掉分支信息。
如果强制禁用Fast forward模式，Git就会在merge时生成一个新的commit，这样，从分支历史上就可以看出分支信息。
禁用Fast forward模式：$ git merge –no-ff -m “merge with no-ff” dev

Bug分支

储藏工作现场：$ git stash
恢复工作现场：$ git stash apply
删除stash内容：$ git stash drop
恢复工作现场并删除stash内容：$ git stash pop

Feature分支

强制删除未合并的分支：$ git branch -D feature-vulcan

多人写作

查看远程仓库的信息：$ git remote
查看远程仓库的信息（详细信息）：$ git remote -v
在本地创建和远程分支对应的分支：$ git checkout -b branch-name origin/branch-name
建立本地分支和远程分支的关联：$ git branch –set-upstream branch-name origin/branch-name
在本地推送分支：$ git push origin branch-name
如果推送失败，先抓取远程的新提交：$ git pull

Rebase

把本地未push的分叉提交历史整理成直线：$ git rebase
rebase的目的是使得查看历史提交的变化时更容易，因为分叉的提交需要三方对比

标签管理

Git的标签是版本库的快照，其实它就是指向某个commit的指针（分支可以移动，标签不能移动）
默认标签都是打在最新提交的commit上

创建标签

创建标签：$ git tag
指定commit_id创建标签：$ git tag commit_id
创建带有说明的标签：$ git tag -a -m “version 0.1 released” commit_id
查看所有标签：$ git tag
查询出的标签不是按时间顺序列出，而是按字母排序的
查看标签信息：$ show
标签总是和某个commit挂钩，如果这个commit既出现在master分支，又出现在dev分支，那么在这两个分支上都可以看到这个标签

操作标签

创建的标签都只存储在本地，不会自动推送到远程，所以，打错的标签可以在本地安全删除
删除本地标签：$ git tag -d
推送某个标签到远程：$ git push origin
推送所有标签到远程：$ git push origin –tags
删除远程标签需要先删除本地标签，在进行远程删除：$ git push origin : refs/tags/

GitHub和码云

使用多个远程库时，Git给远程库起的默认名称是origin，如果有多个远程库，需要用不同的名称来标识不同的远程库

自定义Git

Git显示命令行颜色：$ git config –global color.ui true

忽略特殊文件

有些时候，某些文件放到Git工作目录中，但又不能提交它们，比如保存了数据库密码的配置文件，每次【git status】都会显示未提交文件。
忽略特殊文件：在Git工作区的根目录下创建一个特殊的.gitignore文件，然后把要忽略的文件名填进去，Git就会自动忽略这些文件，.gitignore文件要放到版本库中。
GitHub配置文件：https://github.com/github/gitignore
强制添加被忽略文件到Git：$ git add -f
查看忽略规则：$ git check-ignore -v

配置别名

【git status】：$ git config –global alias.st status
【git checkout】：$ git config –global alias.co checkout
【git commit】：$ git config –global alias.ci commit
【git branch】：$ git config –global alias.br branch
–global参数是全局参数，也就是这些命令在这台电脑的所有Git仓库下都有用，如果不设置，就只对当前的仓库起作用。
将暂存区的修改撤销掉，放回工作区：$ git config –global alias.unstage ‘reset Head’
则：$ git unstage filename == $ git reset HEAD filename
显示最近一次的提交：$ git config –global alias.last ‘log -1’
显示提交的信息：$ git config –global alias.lg “log –color –graph –pretty=format:’%Cred%Creset -%C(yellow)%d%Creset %S %Cgreen(%cr) %C(bold blue)<%an>%Creset’ –abbrev-commit”
Git的全局配置文件放在用户目录下的.gitconfig文件中，仓库的配置文件放在.git/config文件中
